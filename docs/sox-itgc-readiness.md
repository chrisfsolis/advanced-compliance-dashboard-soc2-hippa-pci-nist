# SOX ITGC Readiness

## Purpose

This note documents how the demo dashboard represents SOX ITGC Readiness as a synthetic IT General Controls readiness workflow for systems supporting financial reporting.

SOX coverage is limited to synthetic ITGC readiness workflows and does not represent SOX compliance, financial audit assurance, or control effectiveness validation.

## What SOX ITGC readiness means in this demo

SOX ITGC readiness in this repository means showing synthetic control coverage, evidence completeness, owner review status, and gap visibility for ITGC themes commonly associated with financial reporting systems.

## Included control areas

- Access Management
- Privileged Access
- Joiner / Mover / Leaver
- Change Management
- Deployment Approval
- Segregation of Duties
- Audit Logging
- Backup and Recovery
- Incident and Change Evidence Retention
- Financial Application Owner Review

## Synthetic evidence examples

Synthetic evidence examples include access review exports, reviewer sign-offs, exception lists, remediation notes, privileged user lists, access justifications, change tickets, deployment logs, rollback plans, role matrices, sample log exports, restore test summaries, and owner review checklists.

## Roles and reviewers

The synthetic SOX ITGC workflow uses these demo roles: Finance Application Owner, Internal Controls Reviewer, IT Operations Owner, Security Reviewer, Change Manager, Audit Liaison, and System Owner.

## What this does not do

This project does not determine SOX compliance, does not validate control design or operating effectiveness, and does not provide financial audit assurance.

## Limitations and non-claims

- The control set is synthetic and deterministic.
- The workflow does not connect to production financial systems.
- The evidence examples are placeholders and do not contain customer data.
- The dashboard provides readiness visibility only and does not replace legal, accounting, auditor, or internal controls review.
